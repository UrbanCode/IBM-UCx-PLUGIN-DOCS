
# DevOps-Plan for  DevOps Deploy

DevOps Plan is a planning and tracking tool used to manage deployment-related work items such as plan items, sprints, releases, and generic records. It helps teams organize delivery tasks, update progress, and maintain alignment between development and deployment processes.

The DevOps Plan plug-in provides integration capabilities, allowing you to create and update plan items, modify associated records, track sprint progress, handle release details, and manage any custom record type defined within DevOps Plan.

# Available Steps

# Create Plan Record
Create a new Plan Record in DevOps Plan.This may represent a Release,Project,Work Item,Sprint.

# Update Sprint
Update details of an existing Sprint such as name, dates, description, or any configured sprint fields.

# Update WorkItem

Modify the fields of an existing Work Item within DevOps Plan. Useful for updating status, assignment, effort, or custom fields.

# Update Release

Update Release record details including version,timelines, status, and other metadata associated with a release cycle.

# Update Any Record
Update any DevOps Plan record type .This provides flexibility to update fields for any supported record.

# Delete Record
Delete an existing record from DevOps Plan.You can remove Plan Items, Releases, WorkItems, Sprints, or any custom record.
